# How Do We Work

Source: https://www.notion.so/soupgrowth/How-Do-We-Work-1bf220332c298080b2dacabc27cb1042

We use a three-layered system to organize how work gets done across the business.

## 🗺️ System Maps — The Big Picture (Strategy Layer)

**Format:** Visual — Miro flowcharts or similar.

System Maps show how big processes connect to achieve a goal. High-level view of how different workflows work together — the full journey from start to finish.

**Example: Appointment Angel Hiring — System Map**
- Job ad creation
- Candidate outreach
- Screening
- Trial task
- Final interview
- Onboarding

Some of these will have their own Process Path that breaks it down further.

## 🧭 Process Paths — The Workflow (Execution Layer)

**Format:** Flowcharts, checklists, or step-by-step instructions.

Process Paths sit under System Maps and show the exact steps for a specific part of the process. This is where we go from "what needs to happen" to "how we do it."

**Example: Trial Task Process Path — Appointment Angel Hiring**
1. Send trial task template
2. Set deadline and expectations
3. Review completed task
4. Score against hiring criteria
5. Move forward or reject

Each step may link to more detailed Action Steps.

## 🥾 Action Steps — The Tasks (Hands-On Layer)

**Format:** SOPs, checklists, or step-by-step task guides.

Action Steps are the smallest pieces — the actual tasks or instructions. "Do this, click here, send that." These live inside Process Paths and ensure consistency.

**Example: Send Trial Task — Action Step**
- Open candidate's profile in ClickUp
- Paste trial task message from template doc
- Set due date
- Notify them via email + ClickUp comment

## 👤 Ownership & Self-Sufficiency

Each System Map and Process Path has a designated **owner** — responsible for keeping it up to date and acting as the **main point of contact**.

Everything is documented so everyone can work with **self-sufficiency** and confidence. Whether you're new or experienced, the system helps you move fast, stay consistent, and do your best work.

## 🧩 How It All Fits Together

> **System Map** = The full strategy (e.g. Hiring Journey)
> → **Process Path** = The workflow for one part (e.g. Trial Task)
> → → **Action Step** = The specific task (e.g. Send Trial Task)
